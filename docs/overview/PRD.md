# Product Requirements Document (PRD) — SantriCard

---

## 1. Tujuan Produk
Sistem manajemen kartu untuk santri agar bisa bertransaksi tanpa uang tunai di lingkungan pondok pesantren.

## 2. Fitur Utama
- Generate & reissue kartu (QR Code)
- Top-up saldo virtual
- Transaksi jual-beli via scan QR
- Limit harian transaksi (Rp 20.000)
- Settlement dana pedagang
- Monitoring histori oleh orang tua

## 3. Scope MVP (Minimum Viable Product)
- Backend API (Laravel 13)
- Web App Admin (Next.js + Tailwind)
- Web App Kasir Pedagang (Next.js + Tailwind)
- Web App Monitoring Orang Tua (Next.js + Tailwind)
